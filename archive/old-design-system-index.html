Chat — Spacing rules

Chat New

How vertical gaps between markdown elements are calculated. Each element has a marginBottom and marginTop — the gap is the sum of the bottom element's marginBottom and the top element's marginTop.

PairMathGapRole
body → body 12 + 0 12px Paragraph rhythm
body → H2 12 + 14 26px Section break
body → H3 12 + 10 22px Subsection break
H2 → body 6 + 0 6px Tight anchor
H3 → body 6 + 0 6px Tight anchor
body → bullets 12 + 2 14px List intro
H2 → bullets 6 + 2 8px Heading to list
bullets → H2 12 + 14 26px Section break
bullets → body 12 + 0 12px List to text
body → HR 12 + 14 26px Above line
HR → H2 16 + 14 30px Line to section
HR → body 16 + 0 16px Line to text
bullets → HR 12 + 14 26px List to line
How to read this: "body → H2 = 12 + 14 = 26px" means a paragraph's marginBottom (12) plus H2's marginTop (14) creates a 26px gap. This additive model means spacing is predictable regardless of which elements appear in sequence.

Chat — User bubble

Chat New

Proposed spec for user message bubbles. Fixes the cramped padding (was 2px) and adjusts font size to 18px to match the new body text style.

8:15
Good morning. You slept 7.5 hours with an HRV of 42 ms.
How did I sleep last night?
Recovery looks solid. Resting heart rate was 52 bpm.
What about my heart rate?
Ok thanks

Spec

PropertyValue
Background#FFFFFF
Max Width302px
Border Radius22px (fixed)
Padding10px 16px
Font Size18px
Line Height26px
Letter Spacing0.2px
Margin Right12px
Bubble Gap16px
What changed (vs current):
Padding: 2px → 10px vertical
Font size: 20px → 18px
Line height: 30px → 26px
Letter spacing: 0.4 → 0.2
Border radius: ~23px → 22px fixed
Bubble gap: 20px → 16px
Why:
2px padding made text touch the bubble edge. 20px font in 302px max-width caused aggressive line wrapping on longer messages. The new spec feels more breathable and proportional.
