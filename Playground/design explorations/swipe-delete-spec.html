Swipe-to-Delete Interaction

Ottavia Food Log · React Native · v1.0 · March 2026
1 Overview

Interaction Summary

Users can swipe a food log row left to reveal a delete button. Two ways to delete:

A. Partial swipe + tap — Swipe left to reveal the delete button, then tap it.

B. Full swipe — Swipe all the way left in one gesture to immediately delete.

Reference: iOS Voice Memos swipe-to-delete (see attached video).
Ottavia simplification: single action only (Delete). No "More" or "Move" buttons.

3 Gesture Behavior

Swipe Gesture Gesture

Parameter Value Notes
Direction horizontal, left only Right swipe should not trigger actions
Activation threshold 10px Min horizontal distance before gesture activates
Reveal snap point 74px from right Delete button width (58) + padding (8+8)
Reveal threshold 37px (50% of snap) If swipe > 37px, snap to revealed; else snap back
Full-delete threshold 60% of row width Swiping past this triggers immediate delete
Elasticity 0.8 Resistance factor when swiping past snap point
Velocity threshold 500 px/s Fast flick left triggers full-delete regardless of distance

Dismiss Behavior

Tap elsewhere: If row is in Revealed state and user taps outside the delete button, snap back to Idle.

Scroll: If user starts scrolling vertically while a row is revealed, snap it back to Idle.

New swipe: If user swipes a different row, close the currently revealed row first.

Only one revealed at a time.

5 Animation Timing

All Transitions Animated

Transition Duration Easing Details
Snap to Revealed 350ms spring(damping: 20, stiffness: 180) Row translateX to -74px. Delete button scales from 0 → 1 with overshoot.
Snap back to Idle 300ms spring(damping: 25, stiffness: 200) Row translateX back to 0. Delete button scales to 0.
Delete button bounce 400ms spring(damping: 12, stiffness: 150) Scale: 0 → 1.1 → 0.95 → 1.0. Plays when snap completes.
Row slide-out 250ms easeIn (0.4, 0, 1, 1) Row translateX to -(screen width). Accelerates out.
Row collapse 350ms easeOut (0.25, 1, 0.5, 1) Row height 67px → 0px. Starts 50ms after slide-out begins. Neighboring rows animate up.
Full-swipe delete 200ms easeIn Row continues sliding left off screen. Delete button stretches to fill row width.
Key principle: All gesture-driven animations use react-native-reanimated spring/timing on the UI thread. The Lottie animation is an optional enhancement for the delete button icon reveal only.

Spring Config (Reanimated) Code

// Snap to revealed position const revealSpring = { damping: 20, stiffness: 180, mass: 1, overshootClamping: false, }; // Snap back to idle const snapBackSpring = { damping: 25, stiffness: 200, mass: 1, overshootClamping: true, }; // Delete button scale bounce const bounceSpring = { damping: 12, stiffness: 150, mass: 0.8, overshootClamping: false, }; // Row collapse const collapseTiming = { duration: 350, easing: Easing.bezier(0.25, 1, 0.5, 1), };

7 React Native Implementation

Dependencies

npm install react-native-gesture-handler react-native-reanimated lottie-react-native

Component Structure

// SwipeableRow.tsx import { Gesture, GestureDetector } from 'react-native-gesture-handler'; import Animated, { useSharedValue, useAnimatedStyle, withSpring, withTiming, runOnJS, interpolate, } from 'react-native-reanimated'; const REVEAL_WIDTH = 74; // px to reveal const REVEAL_THRESHOLD = 37; // 50% snap threshold const DELETE_THRESHOLD = 0.6; // 60% of row width const VELOCITY_TRIGGER = 500; // px/s for fast flick const ROW_HEIGHT = 67; // px function SwipeableRow({ item, onDelete }) { const translateX = useSharedValue(0); const rowHeight = useSharedValue(ROW_HEIGHT); const btnScale = useSharedValue(0); const panGesture = Gesture.Pan() .activeOffsetX(-10) // activate after 10px left .failOffsetY([-5, 5]) // fail if vertical .onUpdate((e) => { translateX.value = Math.min(0, e.translationX); btnScale.value = interpolate( Math.abs(translateX.value), [0, REVEAL_WIDTH], [0, 1], 'clamp' ); }) .onEnd((e) => { const shouldDelete = Math.abs(e.translationX) > rowWidth * DELETE_THRESHOLD || e.velocityX < -VELOCITY_TRIGGER; if (shouldDelete) { // Full-delete: slide out + collapse translateX.value = withTiming(-rowWidth, { duration: 250 }); rowHeight.value = withTiming(0, collapseTiming, () => { runOnJS(onDelete)(item.id); }); } else if (Math.abs(e.translationX) > REVEAL_THRESHOLD) { // Snap to revealed translateX.value = withSpring(-REVEAL_WIDTH, revealSpring); btnScale.value = withSpring(1, bounceSpring); } else { // Snap back translateX.value = withSpring(0, snapBackSpring); btnScale.value = withSpring(0, snapBackSpring); } }); // ... animated styles and render }

8 Edge Cases & Accessibility

Edge Cases

ScenarioBehavior
Last item in sectionSection header remains. If section becomes empty, collapse section too (300ms).
Rapid consecutive deletesEach row animates independently. No queueing needed.
Swipe during scrollVertical scroll takes priority. Horizontal swipe fails if >5px vertical movement detected first.
Swipe on expanded player rowSame swipe behavior as normal rows (see iOS reference video at 27-33s).
Offline / API failureRow slides back to Idle with snapBackSpring. Show toast error.
UndoShow toast with "Undo" action for 5s after delete. Re-insert row with reverse collapse animation if tapped.

Accessibility

FeatureImplementation
VoiceOveraccessibilityActions={[{name: 'delete', label: 'Delete'}]}
Haptic feedbackLight impact when snapping to Revealed. Medium impact on delete trigger.
Reduce MotionIf AccessibilityInfo.isReduceMotionEnabled, skip bounce and use withTiming(300ms) instead of springs.
Delete confirmationNot required for swipe delete (same as iOS standard). Undo toast serves as safety net.
