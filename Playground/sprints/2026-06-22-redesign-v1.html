Goal

Primary goal: prepare the app for experts. This sprint lays the visual foundation that the experts surfaces (profile, management, hub) will eventually sit on. The experts buckets themselves are not in the 2-week scope, they come later.

Alongside that, we are refreshing the app's look and feel on a clean-break design language, and reflecting feedback from the 100 initial testers. Visual refresh runs in parallel to production. Current testers stay on the live build. This work ships as v2 a few weeks after they re-engage.

Hard rule: this is a re-skin, not a system rebuild. Existing components get new tokens and new visual treatment. We do not invent new components, change information architecture, or rework patterns. If a surface needs a new component to look right, it goes on the v2 backlog.

What's changing

Foundation, touches every screen

  • Color palette, clean break, new tokens
  • Typography, separate workstream, sweeps all 54 screens
  • Spacing scale
  • Border radii

Anchor surfaces, fully re-skinned

  • Onboarding, 3 buckets
  • Calendar, 3 buckets
  • Chat, 9 buckets
  • Experts, 3 buckets Later

All other 50 screens get the typography sweep this sprint. Color + spacing + radius pass comes surface by surface in later sprints. The Experts surfaces are flagged for later, beyond this 2-week scope.

Canonical visual source. Vika mocks every screen and state here first. Engineers do not start a bucket until its Figma frame is approved.
DESIGN.md (new file, created day 1)
Lives in ottavia-client/DESIGN.md. Vika writes/updates the relevant section the same day she finalizes a Figma frame. Reviewers check PRs against it. No PRs merge to redesign/v1 without DESIGN.md updated.
Playground
Lives in ottavia-design/Playground/. Vika ports stabilized tokens + components into the HTML so FE has a live, copy-paste-able reference. Updated weekly, not per change.
Code
Production truth. Branch redesign/v1 off develop. Parallel to current production. Each PR cites the DESIGN.md section it implements.

Daily rhythm: 15-min standup at 9am. 15-min end-of-day handoff at 5pm.

Out of scope, scope guard
  • New components or new screen templates
  • IA / navigation changes
  • New chat patterns or response types
  • Health-data detail screens (sleep, HRV, nutrition detail, body comp)
  • Profile, settings, goals editing
  • Dark mode (unless tokens give it for free)
  • New animations beyond what's already in code
  • Backend changes for system messages or reports
  • New voice input or composer flows
  • Color + spacing pass on non-anchor screens (typography only this sprint)
