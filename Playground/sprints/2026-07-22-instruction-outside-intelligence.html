The verdict
Every pillar of our sprint is independently mirrored by at least one verified company practice. Voice single-source → Oura's "guardian of AI voice" role and OpenAI's spec-as-code. Binary golden cases from real conversations → Intercom's Batch Test. Deterministic grading before LLM-judge → Sierra's DB-state grading. Prompt PRs + CI gates → OpenAI's versioned Model Spec and Intercom's backtest gate. pass^k long-run testing → Sierra invented the metric. This is convergent validation, not coincidence — and the pages below add the specific refinements worth stealing.
The 10 most transferable practices

Each attributed to the company that verifiably does it. Every one is grounded in a claim that survived 3-vote adversarial verification.

1
One layered instruction spec with named authority levels and an explicit override rule: primer beats agent prompt beats contextual guideline, with a defined conflict default ("when two top-level principles conflict, default to inaction").
OpenAI · chain of command
2
The spec lives in git with a CHANGELOG and dated releases; every change is a diff-reviewable PR. OpenAI ships its behavior spec exactly like software — 5 dated releases in under a year.
OpenAI · model_spec repo
3
Voice/tone defaults sit at the lowest authority layer with a documented implicit-override mechanism — so sensitive contexts (grief, crisis) can legitimately deviate without breaking the spec.
OpenAI · guideline level
4
Decompose prompts into shared, conditionally-injected modules instead of master strings. Claude Code ships 500+ prompt components — tools, subagents, utilities — assembled per context.
Anthropic · Claude Code
5
Ship prompt changes at high frequency, logged per release. Claude Code's instructions changed in nearly every one of 243 releases. Velocity is safe when gated — the gate, not change scarcity, is the safety mechanism.
Anthropic · 243 releases
6
Grade deterministically on end-state ground truth before any LLM-judge — check what the agent DID (database state, tool calls), "not how polished or convincing the conversation sounds."
Sierra · τ-bench grading
7
The policy/instruction doc is a first-class versioned artifact of each eval domain — policy.md checked into git beside the tasks and data it's tested against.
Sierra · policy.md
8
Measure pass^k across repeated identical trials, not pass@1. Expect roughly half your single-shot score by k=8 — GPT-4o fell from ~61% to ~25% on τ-retail. Single-shot evals overstate reliability ~2x.
Sierra · invented pass^k
9
Every prompt change goes backtest → controlled rollout, decided on 2-3 fixed metrics (Fin: resolution rate, CSAT, latency). No change ships on vibes.
Intercom · Fin pipeline
10
~50-question batch suites generated from REAL user conversations, graded Good/Acceptable/Poor with a fixed root-cause taxonomy that explicitly names "Tone wasn't right" as a failure mode — then fix and re-run. Ratings never silently train the agent; a human applies the fix.
Intercom · Batch Test
+
Bonus: defensive prompting — protective instructions embedded wherever the LLM reads end-user input, as standard prompt-injection hygiene.
Intercom
Company evidence, ranked

Ranked by verified evidence quality. Notion, Duolingo, Perplexity and Cursor produced no claims that survived verification — absence of evidence, not evidence of absence.

OpenAI — the instruction architecture benchmarkPrimary sources
  • Model Spec = single cross-product source of truth with an explicit authority hierarchy (Root > System > Developer > User > Guideline) and deterministic conflict rules. Publicly admits the gap between spec and production behavior and ships open compliance evals tracking it.
  • Tone codified as overridable guidelines, not laws — "speak like a pirate" implicitly overrides "avoid swearing." Style rules bind by default, deviate by context.
model-spec.openai.com · github.com/openai/model_spec · alignment.openai.com/model-spec-evals
Sierra — the eval methodology benchmarkPeer-reviewed
  • τ-bench family (ICLR 2025): agents graded on whether the right database updates happened, against a versioned policy.md, with LLM-simulated users. No subjective judging in the core loop.
  • pass^k (all k trials must succeed) as the reliability metric — and eval realism that grows toward production: dual-control users with their own tools, 195K-token knowledge corpora, degraded voice channels.
sierra.ai/blog/benchmarking-ai-agents · github.com/sierra-research/tau2-bench · arXiv:2406.12045
Intercom (Fin) — the production ops benchmarkFirst-party docs
  • Backtest gate + A/B on every model or prompt change, decided on resolution rate, CSAT, latency. Defensive prompting wherever user input reaches the LLM.
  • Batch Test productized: 50-question suites from real conversations, Good/Acceptable/Poor human grading, fixed root-cause taxonomy including "Tone wasn't right" — closest public analog to our per-agent golden sets.
intercom.com/blog/data-privacy-security-ai-chatbots · intercom.com/help — Batch Test docs (May 2026)

Google Health (Fitbit / PH-LLM) — the published methodologyNature Medicine
  • 857 expert-written case studies as golden standards — with human-only holdouts written from scratch, no LLM assistance. 15-question rubric anchored to named clinical frameworks (RU-SATED, SMART). Fine-tuned judge matched human inter-rater agreement; a judge calibrated on one consistent rater beat one trained on all raters pooled.
  • "Recommendations" was the weakest section for models AND experts — advice quality, not knowledge, is the frontier (the model beat human experts on fitness MCQs 88% vs 71%). Production coach ships as labeled preview behind Premium; famous failure mode: told a user to ditch their dog — ignoring a stated life constraint.

Seven refinements worth adopting now

Things the research adds that are NOT yet in the sprint plan — each small enough to fold in this sprint.

1 · Human-only holdout in every golden set
Google kept 50 holdout case studies per vertical written by experts from scratch, no LLM assistance — only training examples were model-drafted-then-rewritten. Our golden sets should mark which cases are pure-human; those are the honest yardstick.
From Google PH-LLM · lands in bucket 14
2 · One canonical rater per agent, not averaged verdicts
Google's judge calibrated on a single low-variance rater beat one trained on all raters pooled. When Vika and Elad disagree on a verdict, don't average — assign the agent to one canonical rater and rewrite the criterion.
From Google PH-LLM · lands in bucket 15
3 · "Never advise removing a stated life constraint"
Fitbit's coach told a user to ditch their dog to walk faster. Add the rule verbatim to the shared primer and back it with eval cases (dog walks, shift work, kids' schedules, injuries). Optimization-over-empathy is the failure mode users punish hardest.
From Fitbit reception · lands in primers + golden cases
4 · Root-cause taxonomy on every failed case
Intercom forces every "Poor" rating into a fixed taxonomy that names tone as a failure mode. Our transcript mining should use a fixed category list from day one — it's what makes failure counts comparable across weeks.
From Intercom Batch Test · lands in bucket 13
5 · Framework-anchored rubrics
Google anchored rubrics to RU-SATED and SMART; graders and prompts shared a vocabulary. Anchor ours to the named frameworks in the methods system so "good sleep advice" is defined once.
From Google PH-LLM · lands in scoring.md
6 · Authority levels + conflict default in the prompt anatomy
Our anatomy template (bucket 04) should name its layers' authority order — primer beats agent prompt beats per-turn context — and define the conflict default, OpenAI-style. Voice sits at the overridable layer with documented sanctioned deviations (grief, crisis).
From OpenAI Model Spec · lands in bucket 04
7 · Weight evals toward Recommendations, not facts
PH-LLM beat human experts on knowledge MCQs but lagged on realistic, achievable recommendations — the same place our low "value" and "lens" criteria point. Golden cases should over-sample "what should I do" turns vs "what is X" turns.
From Google PH-LLM · lands in buckets 14, 10
Honest caveats
Only 4 of the 7 targeted instruction-discipline companies produced claims that survived 3-vote verification — Notion, Duolingo, Perplexity, and Cursor yielded nothing verifiable, which reflects evidence scarcity, not absence of practice. The Anthropic evidence is a third-party mechanical extraction of shipped code (proves the artifact, not the internal process; one claim verified 2-1). Intercom's pipeline claims are first-party self-description corroborated by one VC write-up. Sierra's evidence describes its published research benchmark, not necessarily its internal production pipeline. Oura's eval practice is evidenced by job postings — mandate and intent more than audited operation. Google's PH-LLM methodology is peer-reviewed, but Google explicitly disclaims that the follow-up agent paper describes any shipping product, and the PH-LLM→Fitbit-coach lineage is implied, not confirmed. All figures are snapshots as of Jul 22, 2026.
