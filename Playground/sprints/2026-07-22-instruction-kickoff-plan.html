Five findings that change the plan

From tonight's line-level read of every instruction file, the classifier, the primers, and the grader.

1 · We grade agents on rules we never gave them
The word ceilings (80–250 words by response type) and the emoji policy exist only in eval/standards — no runtime prompt carries them. The General agent is even told "there's no fixed template" while the judge enforces a hard 100-word cap on simple questions. Concision is a top-4 low criterion (22/99). Fix is ~40 words added to VOICE_FORMAT_RULES. Cheapest points in the whole sprint.
2 · The Gateway's 2,722-word prompt is mostly dead code
Since hybrid dispatch, the Gateway only runs on the AMBIGUOUS fallback path — single- and multi-agent turns bypass it. Yet it still ships full Turn Classification + 10-step Routing Logic (~1,900 words) duplicating the Classifier, defines a different intent list than the Classifier's schema (which itself lists intents inconsistently in two places), and routes to a coach_cue tool that isn't attached to it. This is drift you can measure: misrouting is 14% of failure mass.
3 · ~15 voice rules are mechanically checkable, and each is stated 3–4 times
Em-dash, Oxford comma, unit spacing, highlight/bold counts, header levels, banned phrases, word ceilings, date style, status-marker shape. Tonight they became voice_lint.py (built, tested, in the working tree). Every rule the linter owns can shrink in the prompts, and the judge stops wasting attention on them. The Nutrition prompt itself contains two literal em-dashes while ordering "No em-dashes."
4 · Multi-intent is half-built already
The Classifier returns an array of agents and parallel dispatch runs them concurrently — "log 2 eggs and how's my protein?" already fans out. What's missing: intent is a single field (a logging+insight message gets one label and one warmth decision for both branches), and nothing grades whether the merged reply reads as one person. That's the perceived-value gap, and it's testable.
5 · Five agents fly blind, and one shouldn't be an agent
Reports, Goals, Profile, Condition, Engagement have neither golden replies nor test plans. Meanwhile Onboarding is a verbatim template with an if/else — an LLM call that should be code. Profile carries a 200-word routing table that duplicates the Classifier's job. Deleting work is also a quality strategy.

5-line script Instructions version from content hash
Never hand-bump a frozen "v1" again — derive it, and fail CI when instruction files change without the derived version being committed: 
HASH=$(cat src/agent/**/*instructions.ts src/agent/shared/primers.ts | shasum -a 256 | cut -c1-8)
grep -q "$HASH" src/agent/shared/instructionsVersion.ts || {
  echo "instruction files changed: set INSTRUCTIONS_VERSION = \"v1-$HASH\""; exit 1; }
CI draft Prompt-PR gate (GitHub Action)
Trigger on PRs touching src/agent/**: run voice_lint over the golden replies rendered by the changed prompts, run the judge on a fixed 50-case sample, fail the check on lint errors or score regression > 2 points vs the recorded baseline. Braintrust/promptfoo can replace this later; a plain Action is enough to start and adds zero new vendors.
Claude Code Transcript open-coding assistant
Reading 100 transcripts is the sprint's biggest time cost. Split it: Claude Code pre-tags each transcript with a proposed failure category + the first upstream failing turn; you and Elad approve/correct in the 45-minute reading blocks instead of starting from zero. Human verdicts stay the ground truth — the assistant only removes the blank-page cost. Same pattern later drafts golden cases from each confirmed failure for your yes/no.
